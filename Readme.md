# RL_sector

# Optimization thought: 
Using existing sector centroids as part of initial solution and then using Voronoi to generate boundaries for each centroid instead of 
using the whole existing sectoization (since we think current sectors are not optimal) And this method can help avoiding the problem of 
a too huge state , since we need to consider every waypoints), cause in this way only centroids' coordinations need to optimised (corresponding boundaries of sectors generated by voronoi in each step)

# RL structure
1. With continuous state and action space, so I chose PPO algorithm, and available in stable baselines (main.py)
2. Customize an environment registered in gym (I did this accroding to a gym document)(gym_foo/envs/foo_env.py)
