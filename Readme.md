# RL-DAS: Reinforcement Learning-based Dynamic Airspace Sectorization

Code and pre-trained model for the paper "A Scalable Reinforcement Learning-based Approach to Dynamic Airspace Sectorization".

The framework uses Proximal Policy Optimization (PPO) to dynamically re-sectorize airspace in order to balance air traffic controller (ATCo) workload, with sectors generated by a Voronoi-based scheme that guarantees convex sector geometries. The policy is evaluated on one week of UK airspace traffic (17–23 November 2023) sourced from the VATSIM UK Division network.

- **Code:** this repository
- **Data:** Imperial College London Box (direct download) — see <https://imperialcollegelondon.box.com/s/hqnc24v6k1ywh23vsgu3mt84lu28l5vs>

## Requirements

- Python 3.11 or newer (tested on `<OS, e.g. Ubuntu 22.04 / macOS 14>` with Python `<3.11>`)
- The Python packages listed in `requirements.txt`: `gymnasium`, `stable-baselines3`, `numpy`, `scipy`, and `shapely`. PyTorch is installed automatically as a dependency of `stable-baselines3`.

## Data

The train and test data files are large and are kept out of the plain repository. Obtain the four files and place them in the **RL_DAS** (the same folder as `main.py`); the code opens them by relative filename, so they must be in the directory you run the script from. The four files are:

- `train_flight_data.csv`
- `train_conflict_data.csv`
- `test_flight_data.csv`
- `test_conflict_data.csv`

There are two ways to obtain them.

**Option A — Direct download (recommended; works on any system).**
Download the four files from the following link and place them in the repository root:

<https://imperialcollegelondon.box.com/s/hqnc24v6k1ywh23vsgu3mt84lu28l5vs>

**Option B — Clone with Git LFS.**
The files are tracked with [Git LFS](https://git-lfs.com), so they are downloaded automatically when you clone with a Git LFS–capable client — for example [GitHub Desktop](https://desktop.github.com), or the command line after running `sudo apt install git-lfs` and then `git lfs install` once. A plain `git clone` *without* Git LFS, or the "Download ZIP" button on the repository page, retrieves small text pointer files rather than the actual data; if your CSV files are only a few hundred bytes, use Option A instead.

After obtaining the data, the repository root should contain `main.py`, `PPO_policy.zip`, and the four CSV files.

## Usage

Run all commands from the repository root.

> **Dependencies** are installed automatically when `main.py` starts — no manual `pip install` is needed. To install them manually instead, run `pip install -r requirements.txt`.

### Evaluate the pre-trained policy (default)

```bash
python main.py
```

This loads `PPO_policy.zip` and evaluates it over the full week — 42 four-hour intervals (6 per day across 7 days). For each interval, and aggregated per day, it prints the three workload components reported in the paper (aircraft-proximity workload, sector-crossing workload, and sectorization-dissimilarity workload), the reward, and the per-interval and total wall-clock runtimes.

### Retrain from scratch (optional)

Open `main.py`, uncomment the training block, set `total_timesteps` and any hyperparameters, then run:

```bash
python main.py
```

The trained policy is saved to disk. Ensure the filename it is saved under matches the one loaded by the evaluation block (`PPO_policy`) before evaluating.

## Reproducing the paper's results

Running the evaluation above reproduces the per-interval and daily PPO workload metrics reported for the evaluation week (17–23 November 2023).

Note that because these are learning-based results produced by a stochastic policy, the exact output values will differ slightly from one run to the next.
